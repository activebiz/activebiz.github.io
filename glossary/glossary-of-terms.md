### A

- **Account Type:** This terms is a constant which is provided by ActiveBiz which cannot be changed by anyone. This constant was developed at design time. This is used to identify various types of entities which constructs Chart Of Accounts. Based on these constants certain entity of chart of account will have a specific behaviour. for more details see Account Type.
- **Account Category:**  This terms is a constant which is provided by ActiveBiz which cannot be changed by anyone. This constant was developed at design time. This is used to identify various types of Accounts when creating any account in Chart Of Accounts. Every value helps used to catagories the accounts into fixed segments and can be used while searching from the list of Accounts. This will speedup the process for selection of account from large volume of Chart Of Account's records. For more details see Account Catagories.

### I

- **In Active:** This term is used to indicate whether specified record is Active or In Active. If it is InActive then this record can be used by exsting refferences of other master records or transaction records but cannot be used as refference by new master or transaction records. This will prevent this record from appearing from the list where this record is listed for selection for refference to other master or transaction. It is used to terminate its use when that record has no relivence in the current context. Since it is already used by past transactions we cannot delete it and at the same time we want to prevent it's accidental use in future.

### V

- **Version:** This term is used to identify revisions for a records either it can be any Master  e.g. Customer Account, Income Account, Area Master etc. or any Transaction like e.g. Cash Voucher, Sales Order, Sales Invoice, Bank Voucher etc. Based on the value of Version no one can acertain how many time this record changed value of its one or more property. Generally for every Update Operation no matter how many properties values have been changed this Version No will Increment by 1 no. This is helpful in case of Audit also.
